## معرفی پروژه

**IronPanel** برای مدیریت حرفه‌ای سرویس‌های VPN و Proxy طراحی شده است. این پروژه به مدیر سرور امکان می‌دهد چندین پروتکل را از یک پنل واحد مدیریت کند، برای کاربران کانفیگ بسازد، مصرف و انقضا را کنترل کند، نماینده تعریف کند، لایسنس بفروشد، سابسکریپشن اختصاصی ارائه دهد و از طریق ربات تلگرام فروش و مدیریت را ساده‌تر کند.

این پروژه شامل دو بخش اصلی است:

1. **IronPanel**  
   پنل اصلی مدیریت کاربران، پروتکل‌ها، سابسکریپشن، نمایندگان، تنظیمات سرور، SSL، آپدیت و مانیتورینگ.

2. **LicensePanel**  
   پنل مدیریت لایسنس‌ها، پلن‌های فروش، اتصال لایسنس به سرور، ریست سرور ثبت‌شده، مدیریت مشتری و ربات فروش لایسنس.

## پروتکل‌های پشتیبانی‌شده

IronPanel از چند پروتکل مختلف پشتیبانی می‌کند:

- **OpenVPN**
- **WireGuard**
- **Cisco AnyConnect / Ocserv**
- **Hysteria2**
- **Xray / V2Ray**
- **Telegram MTProto Proxy**

هر کاربر می‌تواند براساس تنظیمات مدیر، یک یا چند کانفیگ فعال داشته باشد.

---

## WireGuard

امکانات WireGuard در IronPanel:

- ساخت کلید خودکار برای کاربر
- اختصاص IP داخلی
- ساخت فایل کانفیگ اختصاصی
- دانلود فایل با نام کاربر، مثل:

```text
username.conf
```

- پشتیبانی از MTU قابل تنظیم
- مقدار پیش‌فرض MTU پروژه:

```text
1280
```

- پشتیبانی از `PersistentKeepalive`
- امکان تغییر MTU از داخل پنل
- مناسب برای شبکه‌های موبایل، CGNAT، مسیرهای ناپایدار و اینترنت‌هایی که مشکل Fragmentation دارند

---

## OpenVPN

امکانات OpenVPN:

- ساخت فایل `.ovpn` اختصاصی برای هر کاربر
- دانلود فایل با نام همان کاربر:

```text
username.ovpn
```

- کنترل وضعیت کاربر
- محدودیت حجم و انقضا
- هماهنگی با صفحه Subscription
- امکان repair و sync از داخل اسکریپت‌های پروژه

---

## Hysteria2

امکانات Hysteria2:

- ساخت کانفیگ اختصاصی برای هر کاربر
- پشتیبانی از لینک‌های سازگار:

```text
hysteria2://...
hy2://...
```

- دانلود فایل YAML با نام کاربر:

```text
username.yaml
```

- دانلود لینک URI با نام:

```text
username-hysteria2.txt
```

- باز کردن خودکار پورت‌های لازم در repair
- اتصال به سیستم مصرف، محدودیت حجم و انقضا

---

## Xray / V2Ray

امکانات Xray:

- پشتیبانی از ساخت کانفیگ‌های VLESS / VMess / Trojan براساس تنظیمات پنل
- پشتیبانی از حالت‌های TLS و Reality در ساختار پنل
- خروجی لینک‌های قابل استفاده در کلاینت‌ها
- دانلود فایل لینک‌ها با نام کاربر:

```text
username.txt
```

- امکان مدیریت inboundها
- امکان sync و repair
- اتصال به سیستم مصرف، محدودیت حجم و انقضا

---

## Telegram MTProto Proxy

IronPanel امکان ساخت پروکسی تلگرام اختصاصی برای کاربران را دارد.

امکانات Telegram Proxy:

- ساخت پروکسی جدا برای هر کاربر
- پورت اختصاصی برای هر کاربر
- Secret اختصاصی برای هر کاربر
- لینک قابل استفاده در تلگرام:

```text
tg://proxy?server=SERVER&port=PORT&secret=SECRET
```

- دانلود فایل لینک پروکسی با نام کاربر:

```text
username-telegram-proxy.txt
```

- محاسبه مصرف هر کاربر براساس پورت اختصاصی
- قطع سرویس کاربر بعد از رسیدن به سقف حجم
- امکان نصب و repair از داخل پنل و اسکریپت‌ها

## دامنه اختصاصی Subscription

در تنظیمات پنل می‌توانید برای لینک‌های ساب، دامنه یا ساب‌دامنه جدا تعریف کنید.

مثال:

```text
sub.example.com
```

بعد از ثبت دامنه، لینک‌های سابسکریپشن به‌جای IP اصلی سرور با همان دامنه ساخته می‌شوند:

```text
https://sub.example.com/s/TOKEN
```

اگر این مقدار خالی باشد، پنل از آدرس اصلی سرور استفاده می‌کند.

## نصب سریع

برای نصب، دستور زیر را روی سرور اجرا کنید:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Unknown-sir/ironpanel/main/install.sh)
```

در طول نصب، اسکریپت اطلاعات اصلی را از شما می‌پرسد، مانند:

- دامنه یا IP پنل
- پورت پنل

- انتخاب و فعال‌سازی سرویس‌ها
- تنظیمات پایه موردنیاز

بعد از پایان نصب، آدرس ورود به پنل نمایش داده می‌شود.

## ورود به پنل

بعد از نصب، پنل معمولاً با یکی از حالت‌های زیر در دسترس است:

```text
http://SERVER_IP:PORT
https://YOUR_DOMAIN
```

نام کاربری و رمز عبور همان مواردی هستند که هنگام نصب وارد کرده‌اید.

## آپدیت پروژه

برای آپدیت از داخل پنل:

1. وارد پنل شوید.
2. وارد بخش **Update Manager** شوید.
3. روی دکمه آپدیت بزنید.
4. مراحل آپدیت و درصد پیشرفت را داخل همان صفحه مشاهده کنید.
5. بعد از پایان آپدیت، پنل را restart کنید.

برای آپدیت دستی روی سرور:

```bash
cd /opt/ironpanel
sudo bash upgrade.sh
```

## Telegram Proxy تک‌پورتی با مصرف جدا

از نسخه 18.6.9، Telegram MTProto Proxy با یک سرویس واحد `ironpanel-tgproxy.service` و یک پورت مشترک اجرا می‌شود. هر کاربر Secret اختصاصی خودش را دارد و IronPanel مصرف را از فایل `usage.json` بر اساس همان Secret/User ID داخل مصرف کل کاربر ثبت می‌کند. بنابراین لازم نیست برای هر کاربر یک پورت جدا باز شود.

لینک هر کاربر به‌شکل زیر ساخته می‌شود:

```text
tg://proxy?server=SERVER&port=6969&secret=USER_SECRET
```

مدیریت این بخش از مسیر زیر انجام می‌شود:

```text
Telegram Proxy Manager → /telegram-proxy
```
