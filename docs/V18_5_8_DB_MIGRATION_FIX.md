# IronPanel v18.5.8 — DB Migration Fix

این نسخه خطای نصب/آپگرید روی دیتابیس‌های قدیمی SQLite را رفع می‌کند.

## مشکل رفع‌شده
در نسخه‌های قبلی، دستور `init-db` قبل از اجرای migration سبک دیتابیس، مدل `Admin` را query می‌کرد. اگر دیتابیس قبلی ستون‌های جدید مثل `admin.enabled` را نداشت، نصب با خطای زیر متوقف می‌شد:

```text
sqlite3.OperationalError: no such column: admin.enabled
```

## اصلاح انجام‌شده
اکنون بعد از `db.create_all()` و قبل از هر query، migration سبک SQLite اجرا می‌شود تا ستون‌های جدید جدول‌های قدیمی اضافه شوند.

## برای سرورهایی که خطا داده‌اند
کافی است نسخه جدید را روی GitHub قرار دهید و همان دستور نصب/آپگرید را دوباره اجرا کنید. دیتابیس قبلی حذف نمی‌شود.
